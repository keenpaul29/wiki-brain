# Retaining Computer Science Knowledge

## Summary

This source proposes a retention workflow for computer science study: get the main idea, implement the data structure or algorithm, create flash cards for important concepts, move on, and review repeatedly over increasing time intervals. It argues that spending many consecutive hours on one topic creates weak reinforcement because it mostly repeats what was learned recently.

The source separates conceptual retention from algorithmic review. For facts and ideas, flash cards work well. For algorithms and data structures, the author suggests reviewing printed code and occasionally reproducing algorithms from memory to test understanding.

## Key Ideas

- Learning requires repetition over time, not one long block of exposure.
- Implementation helps turn abstract CS topics into usable knowledge.
- Flash cards are useful for trivia, definitions, and conceptual prompts.
- Algorithm review should focus on understanding and reconstruction, not rote memorization alone.
