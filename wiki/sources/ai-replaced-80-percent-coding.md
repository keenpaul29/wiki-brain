# AI Replaced 80% of Coding

## Summary

This source argues that AI has automated much of syntax-level coding but has not replaced software engineering. It frames the future engineer as an architect, investigator, reviewer, negotiator, and accountable owner of systems. The article lists seven durable skills: architectural reasoning, distributed debugging, legacy-system archaeology, requirements engineering, strategic system thinking, legal and ethical accountability, and human connection.

Its central claim is that AI raises the bar: writing code becomes cheaper, but validating, directing, integrating, and owning that code become more important. This lines up strongly with the system design source and the wiki workflow sources: the engineer's advantage becomes structured judgment over raw generation speed.

## Key Ideas

- Architecture is business risk management under constraints, not a single correct answer.
- Distributed failures require temporal and causal reasoning that static code generation often misses.
- Legacy code should be understood before being refactored.
- Requirements work is a human negotiation problem.
- AI code still requires human review, liability awareness, and ethical auditing.
