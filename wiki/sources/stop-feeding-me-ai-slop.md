# Stop Feeding Me AI Slop

## Summary

This source argues that AI is valuable as a writing and thinking assistant but harmful when used to simulate understanding. The author focuses on technical communication: design docs, PRDs, code reviews, Slack messages, and implementation notes that look polished but contain little specific judgment from the person who sent them.

The core distinction is between AI-assisted expression and AI-replaced thinking. Good use starts with the human doing the analysis, making decisions, and understanding the context, then using AI to improve prose. Bad use delegates the thinking itself, producing generic documents that reviewers must decode and authors cannot defend.

## Key Ideas

- Human value in technical communication is distillation: concrete experience, decisions, tradeoffs, and context.
- AI-generated design docs can appear complete while skipping real option analysis and constraint handling.
- Generated code and generated runbooks are dangerous when the owner cannot explain them.
- AI amplifies judgment; it accelerates strong engineers and also accelerates weak or absent reasoning.
- The working rule is: think first, write second, and use AI mainly for the second part.
