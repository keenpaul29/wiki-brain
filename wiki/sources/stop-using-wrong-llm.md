# Stop Using the Wrong LLM

## Summary

This source argues against choosing LLMs through generic leaderboards. It proposes mapping work into cognitive lanes and matching models or agent chains to the task, while avoiding classes of work where current models remain unreliable.

## Key Ideas

- The source frames useful LLM selection around cognitive demand rather than a single "best model" ranking.
- It identifies hard failure zones such as implicit chart extraction, multi-column scanned PDFs, directional diagrams, long self-referential reasoning, many-entity tracking, cluttered-image counting, admitting ignorance, and precise long-tail facts.
- Model choice should begin by asking what ability the task needs, then selecting the model strongest in that lane.
- Compound work may need routing across multiple models or agents because different steps have different cognitive demands.
- The practical risk is not only model error, but extra verification time and reputation damage when weak model/task fits are shipped unchecked.
