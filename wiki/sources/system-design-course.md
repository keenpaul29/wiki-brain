# System Design Course

## Summary

This source is a broad curriculum for designing systems at scale. It starts with networking and infrastructure basics, then moves through system components, architectural styles, scaling strategies, reliability patterns, and interview-style case studies such as URL shorteners, WhatsApp, Twitter, and Netflix.

The course presents system design as a requirements-driven process. A good design starts by clarifying functional and non-functional requirements, estimating scale, defining APIs and data models, choosing storage and communication patterns, then identifying bottlenecks and failure modes.

## Key Ideas

- System design defines architecture, interfaces, and data to satisfy requirements.
- Early architectural decisions are hard to reverse, so high-level reasoning matters.
- Core building blocks include DNS, load balancers, caching, CDNs, proxies, APIs, message brokers, databases, replication, sharding, and consistency strategies.
- Large-scale designs require explicit tradeoffs around availability, latency, durability, cost, and operational complexity.
- Case-study practice builds reusable design judgment.
