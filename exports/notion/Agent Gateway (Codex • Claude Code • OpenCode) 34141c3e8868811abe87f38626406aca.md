# Agent Gateway (Codex • Claude Code • OpenCode)

## Mission

Operate this brain as a high-leverage system: capture fast, decide clearly, execute deeply, and compound learning.

## Start Here (Every Session)

1. Open `Auto Priority Queue` (Tasks).
2. Open `Project Priority Radar` (Projects).
3. Commit to top 3 outcomes before doing anything else.

## Power Stack (Canonical)

1. `GitHub Repo Tracker (All Repos)` for lane selection.
2. `Tasks` with weighted scoring and blocked-state visibility.
3. `Projects` with health/stage and weighted scoring.
4. `Knowledge` + `Memory Graph` for compounding intelligence.
5. `Playbooks` + `Decision Log` for repeatable quality.

## Scoring Contract

- Score `Impact`, `Urgency`, `Effort` on 1-5.
- Let `Priority Score` rank work automatically.
- Re-score weekly or after major context shifts.

## Minimum Quality Standard

- Every task has: lane, repo, impact, urgency, effort.
- Every project has: health, stage, review date, score inputs.
- Every strategic change has: a decision log entry.
- Every key learning has: evidence and understanding level.

## Anti-Entropy Rules

- No orphan tasks without repo/lane.
- No active project without review date.
- No scoreless active work for more than 7 days.
- No duplicate trackers as source of truth.

## Agent Compatibility

Codex, Claude Code, OpenCode, and future agents can execute this exact workflow by following this gateway and canonical resources.
